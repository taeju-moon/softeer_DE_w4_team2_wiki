# 요구사항명세서
## JFK 공항 출발 택시 목적지별 수요 분석 및 버스 노선 제안 서비스

---

## 1. 프로젝트 개요

### 1.1 배경
본 프로젝트는 Apache Spark를 활용하여 NYC TLC(Taxi and Limousine Commission) Trip Record Data를 분석하는 기존 개인 과제를 기반으로, 팀 활동 요구사항인 "자율주행차 데이터라면 어떤 Data Product를 만들 수 있을까?"에 대한 아이디어를 구체화한 조별 프로젝트이다.

### 1.2 주제
JFK 공항에서 출발하는 Yellow Taxi의 목적지별 이용량 및 요금 데이터를 분석하여, 수요가 높은 목적지 구간(corridor)을 도출하고, 해당 구간의 실제 MTA 버스 노선 배차 현황과 비교하여 **수요 대비 공급이 부족한 노선을 식별**한다. 이를 데이터 상품(Data Product)으로 정리하여 버스 회사에 판매할 수 있는 형태의 분석 리포트를 제공하는 것을 목표로 한다.

### 1.3 비즈니스 시나리오
> "공항에서 출발하는 택시 수요 데이터를 기반으로, 특정 지역행 수요가 매우 높음에도 불구하고 현재 버스 배차 간격이 이를 따라가지 못하는 노선을 찾아내어, 버스 회사에 배차 증편 또는 신규 노선 신설을 제안하는 데이터 상품"

---

## 2. 프로젝트 범위(Scope)

### 2.1 포함 사항
- JFK 공항에서 출발(Pickup)하는 Yellow Taxi 트립 데이터 분석
- 목적지(Zone)별 이용 건수 및 요금 분포 분석
- 인기 목적지 Zone을 지리적으로 클러스터링하여 노선 후보(corridor) 도출
- 시간대별 택시 수요 패턴(피크 시간) 분석
- 해당 corridor를 실제로 지나는 MTA 버스 노선의 배차 데이터와 비교
- 택시 수요 대비 버스 공급(배차 간격/용량) 갭 분석 및 시각화

### 2.2 제외 사항
- 날씨 조건과 수요 상관관계 분석 (원본 개인 과제 요구사항이며, 본 팀 프로젝트 주제와 무관하여 제외)
- 멤버십 요금제 설계 및 시뮬레이션 (최초 주제 문구에는 포함되어 있었으나, 논의 결과 버스 노선 제안에 집중하는 것으로 스코프 조정하여 제외)
- 버스 운영비(인건비, 유가, 차량 감가상각 등) 기반의 정밀한 손익분기점 계산 (수요-공급 갭 파악 수준으로 단순화)

### 2.3 최종 정정된 주제명 (제안)
> "JFK 공항 출발 택시 목적지별 수요·요금 분석 기반 버스 노선 배차 개선 제안 서비스"

## 3. 데이터 소스

| 데이터 | 출처 | 용도 |
|---|---|---|
| Yellow Taxi Trip Records | NYC TLC 공식 사이트 (2026년 공개된 월, 다운로드 시점 기준) | 트립 건수, 거리, 요금, 시간 분석 |
| taxi_zone_lookup.csv | NYC TLC 공식 사이트 | LocationID → Zone/Borough 이름 매핑 |
| taxi_zones shapefile (위경도) | NYC TLC 공식 사이트 (GIS 데이터) | Zone 중심 좌표 기반 지리적 클러스터링 |
| MTA GTFS Static Data | MTA 공식 GTFS 배포 페이지 | 노선/정류장/배차시간표 (stop_times.txt 등) |

### 3.1 분석 대상 공항
**JFK 공항**으로 확정
- 사유: TLC 데이터 내 픽업 건수가 가장 많아 통계적 유의성 확보에 유리, JFK 고유의 flat fare 구조 보유, 국제선 중심으로 초행 승객 비율이 높아 버스 노선 제안 시나리오와 부합, 실제 유사 비즈니스(공항 셔틀버스)가 이미 존재하는 시장

### 3.2 분석 대상 서비스 타입
**Yellow Taxi만** 사용 (Green Taxi, HVFHV(Uber/Lyft) 제외)

### 3.3 분석 기간
**2026년 데이터** (TLC 데이터 공개 시차(통상 2~3개월)를 감안하여, 실제 다운로드 시점에 공개되어 있는 2026년 월 단위 데이터를 모두 사용)

## 4. 기능 요구사항

### 4.1 데이터 수집 (Data Ingestion)
- TLC 웹사이트에서 2026년 Yellow Taxi Parquet 파일을 월 단위로 다운로드 및 병합
- taxi_zone_lookup.csv 및 taxi_zones 위경도 데이터 로드
- MTA GTFS Static Data(routes.txt, stops.txt, stop_times.txt, trips.txt 등) 다운로드 및 로드

### 4.2 데이터 정제 (Data Cleaning)
- PULocationID 기준 JFK Zone(LocationID=132)으로 필터링
- 결측치 처리 (제거 또는 대표값 대체)
- pickup/dropoff datetime → 표준 timestamp 변환 및 trip duration 계산
- 이상치 필터링: **통계 기준 적용** (IQR 또는 상하위 0.1~1% 백분위 컷오프로 trip_distance, trip_duration, fare_amount 이상치 제거)
- JFK Flat Fare 트립은 별도 구분 없이 일반 fare_amount와 동일하게 취급
- service_zone이 "Airports"인 목적지(타 공항 및 공항 내 이동)는 노선 후보 분석에서 제외

### 4.3 목적지별 이용량 및 요금 분석
- DOLocationID(Zone) 단위로 이용 건수 집계
- 평균 Trip Duration, 평균 Trip Distance 계산
- Zone별 요금(fare_amount) 분포(평균, 중앙값, 분산) 계산
- 파레토 기준(예: 상위 80% 이용 비중을 차지하는) 인기 Zone 추출
- 추출된 인기 Zone들을 위경도 기반으로 지리적 클러스터링(K-means 등)하여 "노선 후보 corridor" 생성

### 4.4 시간대별 피크 분석
- 시간대별(hour of day) 트립 시작 건수 집계
- 피크 시간대 식별 및 시각화 (막대그래프)
- corridor별 피크 시간대 별도 분석 (버스 배차 스케줄 설계 근거로 활용)

### 4.5 버스 수요-공급 갭 분석
- 대상 버스 노선: **Q3, Q10, Q70 SBS, B15** (JFK 관련 주요 4개 노선)
- MTA GTFS Static Data에서 각 노선의 배차 간격(운행 시간표 기준) 산출
- corridor별 택시 수요량(시간대별 트립 건수)과 해당 corridor를 지나는 버스 노선의 배차 간격/용량을 비교
- "택시 수요는 높으나 버스 배차가 이를 따라가지 못하는" 노선 및 시간대를 식별하여 리스트업
- 갭이 큰 순서로 정렬하여 배차 증편이 필요한 우선순위 노선 도출

### 4.6 출력 (Output)
- 중간 처리 결과: **Parquet** 포맷으로 저장
- 최종 리포트/집계 결과: **CSV** 포맷으로 저장 (엑셀 등에서 바로 확인 가능하도록)
- 시각화 결과물: 목적지별 이용량 히트맵/막대그래프, 시간대별 피크 그래프, 요금 분포 히스토그램, 수요-공급 갭 비교 그래프

## 8. 성공 기준

- JFK 출발 택시 데이터에서 통계적으로 유의미한 인기 목적지 corridor가 도출될 것
- 도출된 corridor가 실제 MTA 버스 노선(Q3, Q10, Q70 SBS, B15)과 매핑 가능할 것
- 택시 수요 대비 버스 배차 공급이 부족한 구체적인 노선/시간대가 최소 1개 이상 식별될 것
- 분석 결과가 버스 회사 입장에서 실행 가능한(actionable) 제안 형태로 정리될 것
